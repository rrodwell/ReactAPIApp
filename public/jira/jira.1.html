  Description    

In ServicePoint 2017 R4, iPOS Hold functionality is enhanced by allowing Deferred Orders to be put on Hold and stored in the Store/Recall queue.

In prior ServicePoint releases, Deferred Orders could not be held and stored in this fashion.

Preconditions:
iPOS and Mobile ordering configured for the Lab in use. No additional configuration is needed.

Resolution: Unresolved Votes: 0
Labels: None
Remaining Estimate: Not Specified
Time Spent: Not Specified
Original Estimate: Not Specified

Zephyr Teststep:
Test Step Test Data Expected Result
1

Open the iPOS setup page

Page is opened and all settings switches are disabled
2

Attempt to toggle the cash drawer switch

Switch cannot be toggled while print receipts is disabled
3

Toggle Print Receipts on

Print receipts is enabled and the receipt printer is recognized
4

Toggle Use cash drawer on

Cash drawer is enabled
5

Toggle Print Receipts off

Print receipts and cash drawer are both disabled
6

Toggle Print receipts and Use cash drawer back on then sign into iPOS

User is signed in
7

Enter any item and tender the transaction with a cash tender type

Cash drawer is opened and receipt is printed
8

Attempt to begin a transaction without closing the cash drawer

Attempt is blocked by and error message to close the cash drawer
9

Close the cash drawer
10

Attempt to toggle Print receipts off

Attempt is blocked because the cash drawer cannot be toggled while a user is signed in

  Description    

Disable cash drawer functionality if receipt printer switch is off is an enhancement for R4 2017. There is no BRS, therefore, this test is written on knowledge of prior iPOS experience and WIT 423737.

This test case covers the verification of new functionality added for this enhancement, denoted by the f in the title.

WIT Description: Prevent cash drawer functionality if the receipt printer toggle switch under iPOS Setup is turned off. Cash drawer relies on the printer connection for functionality, so without the printer switch on, cash drawer should never work.

  • Functionality to be verified:
  • Disabling the receipt printer disables the switch for cash drawer and disables all functionality for the cash drawer
  • Cash drawer can not be enabled while receipt printer is not enabled
  • Receipt printer can or cannot be disabled/enabled while the user is signed in?
  • Re-enable receipt printer? Does it auto re-enable the cash drawer? The assumption is no.
  • Cash drawer and receipt printer connectivity - Loss of connection, reconnect, pair when enabled,
  • Cash drawer Status updates

Preconditions:
Epson TM88V Paired to the iPad. Cash drawer plugged into the connected receipt printer. All setup switches are disabled.
